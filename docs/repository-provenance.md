# Repository Provenance

## Canonical public portfolio

`PeterYYong/AIFFEL_quest_rs` is the canonical AIFFEL repository currently used for the public researcher portfolio. The suggested public-facing repository name is `aiffel-applied-deep-learning-portfolio`; changing the GitHub repository slug is a repository-settings action separate from the content restructuring.

## Path migration

Opaque course paths were replaced by descriptive, lowercase paths. Original AIFFEL track/unit identifiers remain in each project README and in [`notebook-index.md`](notebook-index.md). Git commit history preserves the previous paths and review records.

The restructuring also removed:

- zero-byte notebook scaffolds for GoingDeeper GD02's misspelled empty file, GD09, and MainQuest Quest03–Quest05;
- untouched template READMEs for units with no substantive artifact;
- two misplaced GD07 notebook copies under the former GD06 directory.

These removals do not imply completed projects at those unit numbers.

## Original course and peer-review archives

The repositories `AIFFEL_quest_rs-1` through `AIFFEL_quest_rs-8` may preserve course snapshots, peer-review targets, shared artifacts, or contributions by other participants. They are not treated as candidate-authored portfolio projects and should not be used as evidence of sole authorship.

Before any later merge or archive operation:

1. Compare file paths and hashes.
2. Identify the notebook author, provided baseline, peer reviewer, and team contributors.
3. Preserve unique review records and original commit history.
4. Exclude datasets, checkpoints, credentials, and restricted material.
5. Move only clearly attributable candidate work into the curated repository.
